## 1. 产品愿景与核心体验

### 1.1 一句话描述

在「琥珀港」王冠竞拍会上，玩家作为代理人，与三名对手对**未知仓库**进行**实时明拍**，通过情报、估值与心理博弈决定出价，成交后开箱结算，将战利品纳入持久仓库，积累资产与收藏。

### 1.2 核心体验支柱


| 支柱        | 描述                                              |
| --------- | ----------------------------------------------- |
| **信息不对称** | 仓库内 15–40 件藏品，初始仅见轮廓/品质/尺寸等部分信息，需靠随机情报与角色技能逐步揭示 |
| **实时博弈**  | 明拍窗口内可见最高价与领先者，需在倒计时内决定加价、自定义出价或放弃              |
| **风险回报**  | 拍得真值低于出价则亏损；亏损时对手按地图规则获得「福利金」分红                 |
| **资产积累**  | 银币 K 线、多页仓库、红色藏品收藏、排行榜 — 形成单局外的长期目标             |
| **角色差异化** | 五名出战角色技能不同，Bot 性格参数不同，同一仓库不同打法                  |


### 1.3 目标玩家

- 喜欢《仓储拍卖》《竞拍之王》类估值博弈的玩家
- 偏好轻策略、重读信息与心理战的休闲/中度玩家
- 对中世纪幻想 × 二次元角色有审美偏好的手游/独立游戏受众

### 1.4 与竞品差异

- **明拍**（非密封暗拍）：实时可见领先价，强调窗口内决策与秒杀机制
- **配置表驱动**：258+ 道具、地图权重、Bot 参数均可 CSV/JSON 热调
- **世界观统一**：琥珀港、王冠竞拍会、五位代理人角色有完整 lore 与美术方案

## 2. 世界观与叙事框架

### 2.1 时代与地点

**「圣契王庭」崩塌后**，边境商都 **「琥珀港」** 成为各方势力交汇点。王国旧贵族、佣兵团、魔法学院、地下公会与黑甲骑士团争夺遗迹仓库中的古代宝物。

玩家不是传统冒险者，而是被邀请进入 **「王冠竞拍会」** 的**代理人**：每一轮竞拍都是无硝烟的决斗，金币、情报、心理压迫与时机判断决定谁能带走真正有价值的藏品。

### 2.2 世界气质

华丽但危险 — 中世纪宫廷、魔法拍卖行、地下黑市、冒险者公会混合。视觉上有二次元美少女角色的明亮吸引力，同时保留金属、羊皮纸、宝石、暗纹、蜡封和魔法光效。

### 2.3 对局叙事包装

每局开局播放 **遗产 Lore 过场**（`config/match_heritage_lore.json`）：按所选地图随机展示一条仓库背景（名称 + 描述），例如：

- **琥珀商馆**：北门货运仓库、旧王庭清算库、佣兵团战利品车……
- **银雾修道院**：修会赎回仓、祈愿礼器库、疫雾隔离货箱……
- **星塔穹庭**：星塔实验封存箱、观星台坠落残件、占星师遗产包……
- **黑蔷薇王陵**：王陵陪葬封箱、黑甲骑士遗物柜、诅咒商队扣押货……

## 3. 核心玩法循环

```
大厅（选模式/地图/角色）
    ↓ 扣门票（正式局）
选图确认 → 匹配/建房 → 对局 seed 生成仓库
    ↓
开场 CG（拍卖开始 + 遗产 Lore）
    ↓
【循环最多 5 轮，可第 6 轮加赛】
    INFO（随机情报 + 角色技能揭示）
    → OPEN_BOARD（公布面板状态）
    → BID_WINDOW（默认 25 秒出价窗）
    → BID_RESOLVE（取本轮各席 peak bid 最高者）
    → 检查秒杀 / 速购 / 继续
    ↓
成交 → UNBOX（逐件揭示动画）→ SETTLEMENT（盈亏结算）
    ↓
MATCH_END → 领取战利品入仓库 / 回收银币 → 更新 Portfolio K 线 → 回大厅
```

### 3.1 模式入口


| 入口        | 说明                       | 门票     | 盈亏入库 |
| --------- | ------------------------ | ------ | ---- |
| **快速匹配**  | 假匹配 5–10 秒后开局，Bot 补位     | 扣      | 是    |
| **AI 练习** | 单人立即开局                   | 免      | 否    |
| **房间对战**  | ENet 2–4 人，房主设人数，全员准备后开局 | 免（练习流） | 否    |


### 3.2 单局时长预期

- 5 轮 ×（情报 CG + 25 秒出价窗 + 结算动画）≈ **8–15 分钟**
- 含开箱与结算 UI 可达 15–20 分钟

## 4. 对局系统：明拍竞拍

### 4.1 明拍 vs 暗拍（设计演进）


| 维度    | 暗拍（旧/遗留代码） | 明拍（当前）                 |
| ----- | ---------- | ---------------------- |
| 出价可见性 | 轮末才公开      | 实时公开最高价与领先者            |
| 出价次数  | 每轮 1 次密封   | 窗口期内各席提交本轮最高价意向        |
| 结束条件  | 速胜倍率       | 秒杀倍率 / 速购价 / 最大轮次 / 流拍 |


> 注：`auction_rules.gd`、`bot_brain.gd` 等暗拍逻辑仍保留但未接入主流程。

### 4.2 阶段状态机

```
LOBBY → INFO → OPEN_BOARD → BID_WINDOW → BID_RESOLVE
    → （下一 round 或 UNBOX → SETTLEMENT → MATCH_END）
```

**实现**：`scripts/match/match_controller.gd` + `scripts/autoload/game_constants.gd`（`MatchPhase` 枚举）

### 4.3 默认对局参数（`config/match_config.json`）


| 参数                       | 默认值     | 说明                        |
| ------------------------ | ------- | ------------------------- |
| `auction_mode`           | `open`  | 明拍模式                      |
| `seat_count`             | 4       | 2–8 人（当前固定 4）             |
| `player_seat`            | 0       | 主角席位                      |
| `max_rounds`             | 5       | 最大轮次                      |
| `bid_window_seconds`     | 25      | 出价窗口秒数                    |
| `min_raise`              | 50,000  | 最小加价步进（配置项，当前实现未严格强制链式加价） |
| `starting_bid`           | 10,000  | 起拍价参考                     |
| `reserve_quick_buy`      | 0       | 速购价（0 = 不启用）              |
| `bot_tick_seconds`       | 1.2     | Bot 调度参考                  |
| `warehouse_grid_w × h`   | 10 × 20 | 对局仓库网格                    |
| `warehouse_item_min/max` | 15 / 40 | 每场道具件数范围                  |
| `max_place_retries`      | 24      | 摆放失败重试次数                  |
| `unbox_item_delay`       | 0.55    | 开箱逐件延迟（秒）                 |


### 4.4 出价规则

**当前实现逻辑**（窗口式 peak bid，非连续加价链）：

1. 出价窗口内，各席位可多次更新**本轮最高意向价**（锁定出价）
2. Bot 在窗内随机时刻 `[1.0, window_sec]` 提交出价
3. 人类操作：**加价** / **自定义出价** / **本轮放弃** / **整场放弃**
4. 窗口结束：未锁定且 peak ≤ 0 的席位视为 pass；**人类若本轮 peak ≤ 0 → 触发整场弃权**
5. 取各席 **round peak bid** 最高者更新 `OpenAuctionBoard`
6. 校验：`bid > 0`，`bid ≤ seat.silver`，窗口开放（resolve 阶段允许 `allow_closed_window`）

**数据结构 `OpenAuctionBoard`**：


| 字段                    | 说明                 |
| --------------------- | ------------------ |
| `current_highest_bid` | 当前最高价              |
| `current_leader_seat` | 领先席位（-1 = 无）       |
| `min_next_bid`        | 下次最低有效出价           |
| `round_index`         | 当前轮次 1..max_rounds |
| `raises_this_round`   | 本轮加价次数             |
| `window_active`       | 是否在出价窗口            |


### 4.5 结束条件


| 条件       | 触发                         | 结果               |
| -------- | -------------------------- | ---------------- |
| **秒杀**   | 第一名 peak ≥ 第二名 peak × 轮次倍率 | 立即成交             |
|          |                            |                  |
| **封盘成交** | 达 `max_rounds` 且有有效最高价     | 按最高价成交           |
| **加赛**   | 第 5 轮最高价并列                 | 进入第 6 轮（倍率用 1.1） |
| **流拍**   | 全程无有效最高价                   | 无赢家，跳过开箱，全员净收益 0 |


**秒杀倍率**（`config/map_modes.json` → `instant_kill_multipliers`）：

```
第 1 轮 2.0 → 第 2 轮 1.8 → 第 3 轮 1.6 → 第 4 轮 1.4 → 第 5 轮 1.2 → 第 6 轮 1.1
```

**黑骑士特殊规则**（`noctis_blackrose`）：

- **同价胜出**：与他人出价相同时由黑骑士优先胜出
- **秒杀**：黑骑士排名第一时直接进结算（无需等待倍率判定）
- **双黑骑士同价第一**：秒杀失效（联机场景）

### 4.6 结算经济（`scripts/match/settlement.gd`）

**赢家**：

- `delta = -winning_bid`（扣成交价）
- `profit = true_value - winning_bid`

**亏损分红**（成交价 > 仓库真实总值）：

- `loss_amount = winning_bid - true_value`
- 非赢家每人获得：`floor(loss_amount × dividend_rate / 对手人数)`
- `dividend_rate` 由地图模式决定（1.0 / 0.66 / 0.33）

**超价标记**：

- `is_overbid = winning_bid > true_value`
- `GameConstants.OVERBID_COMPENSATION_RATE = 0.1` 已定义但**尚未接入结算**

### 4.7 随机情报系统（`scripts/match/match_intel.gd`）

每轮 INFO 阶段随机抽取一条情报，驱动 UI 揭示仓库部分信息。共 **11 种类型**，带权重与去重（`DEDUP_MAX_ATTEMPTS = 16`）：


| 类型                       | 权重  | 效果概要          |
| ------------------------ | --- | ------------- |
| QUALITY_AVG_CELLS        | 12  | 某品质道具的平均占格    |
| WAREHOUSE_AVG_CELLS      | 10  | 全仓平均占格        |
| RANDOM_SAMPLE_AVG        | 12  | 随机抽样 N 件均价    |
| RANDOM_FULL_INFO         | 10  | 随机 1–3 件全信息揭示 |
| QUALITY_ALL_SIZE         | 11  | 某品质全部尺寸       |
| HIGHEST_VALUE_FULL       | 3   | 最高价值单品全信息     |
| WAREHOUSE_QUALITY_ROSTER | 11  | 各品质件数 roster  |
| WAREHOUSE_META           | 9   | 仓库元信息（总格/件数等） |
| ALL_OUTLINES             | 8   | 全部轮廓          |
| HIGHEST_QUALITY_ONE      | 7   | 最高品质单品提示      |
| QUALITY_TOTAL_CELLS      | 10  | 某品质总占格        |


情报通过 **MatchCinematicOverlay** 以 CG 形式展示，并写入揭示索引供 UI 刷新。

## 5. 仓库与道具系统

### 5.1 对局仓库（局内）

- **网格**：10 × 20（200 格），由 `match_seed` **可复现**生成
- **件数**：15–40 件，加权随机
- **三层抽样**（`docs/04-probability-tables.md`）：

```
match_seed → RNG
  → roll spawn_pool_table（pool_weight）
  → roll items in pool（item weight × quality_modifier × 地图 weight_key）
  → 二维摆放（size_w × size_h）
  → 放不下则 retry（max 24 次）或降级 pool
```

**可见性分层**（揭示状态）：

1. 完全隐藏（仅轮廓）
2. 品质 + 尺寸
3. 部分信息（名称/价值仍隐藏）
4. 完全揭示（名称、品质、尺寸、价值）

### 5.2 道具主表（`config/items_master.csv`）

**当前规模**：258 条（设计目标 500 条，工具 `_tools/generate_items_master.py` 可重新生成）

## 7. Bot 与 AI 行为

**主逻辑**：`scripts/match/open_auction_bot.gd`

### 7.1 出价决策

1. 基于 `ValuationScript.estimate_from_intel`（仓库情报 + `valuation_bias`）估算 low/mid 区间
2. 计算与当前领先价差 → `urgency`
3. 结合 `aggression`、`raise_tendency`、轮次、是否已领先 → 出价概率
4. urgency > 0.65 时出价概率 ≥ 0.88
5. 已领先时降低出价概率

### 7.2 出价金额

- 在 low/mid 估值间插值
- 可能略高于当前最高价 2–12%，或虚标（`bluff_rate`）
- 上限 `mid × max_chase_ratio`，且 ≤ 席位银币

### 7.3 调度

- 窗内 `fire_at` 随机 `[1.0, window_sec]`
- 非人类 Bot 不参与 Portfolio，使用 `bot_bid_cap`

### 7.4 遗留 AI

- `bot_brain.gd`：按 archetype（BLOCKER / INTEL / DISRUPT）调价，供暗拍路径，**当前明拍未调用**
- `match_controller_backup.gd`：旧版控制器备份
